INFECTIONS table

Currently, the schema includes only one table, infections, with the following fields:

  • SOURCE: code describing the source of information
  • DATE: day of the statistics
  • COUNTRY: English name for the country
  • COUNTRYCODE: ISO ALpha-3 code of the country.
  • L1REGION: (optional) wide-area administrative or geographical region within the country. For example a "state" in the United States, a "province" in China or Canada, an oversea territory like Faroe Islands for Denmark or Martinique for France.
  • L2REGION: (optional) mid-area region within a country or an l1region. For example a "region" in Italy or a "county" in the UK.
  • L3REGION: (optional) a small-area region within the country or the l2region, for example an italian "province".
  • CITY: (optional) a city, town or village.
  • TESTED: (optional) number of people tested. Cumulative.
  • INFECTED: (optional) number of confirmed infected. Cumulative.
  • RECOVERED: (optional) number of infected who have recovered. Cumulative.
  • HOSPITALISED: (optional) number of infected who have been hospitalised. Cumulative.
  • HOSPITALISED_ICU: (optional) number of infected in the intensive care. Cumulative.
  • HOMEQUARANTINED: (optional) number of infected with home quarantine. Cumulative.

Note 1: We collect the data as they come from the data sources, we do not check for their consinsentcy nor correctness. It is possible that different sources provide slighlty different statistics.

Note 2: The database is being built and it is likely that the datastructure will change. Check this webpage regularly to be notified about changes.
